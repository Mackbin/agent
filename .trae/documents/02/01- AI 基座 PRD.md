# 一、文档概述

## 1.1 文档目的

本文档为「AI 中间基座」的产品需求文档（PRD），明确该基座的产品定位、核心功能、技术架构及 MVP 实施范围，作为开发、测试、部署的核心依据。

## 1.2 产品定位

无自研大模型底座，完全依赖第三方大模型 API（OpenAI、DeepSeek、阿里通义等），自建轻量化 AI 中台基座。核心定位为「个人可用、渐进式扩展」，采用 MVP 思路分阶段实施，优先实现核心功能，确保快速落地。

## 1.3 适用范围

本文档适用于该 AI 中间基座的开发人员、测试人员、部署人员，明确各环节需求和标准；同时供个人使用者参考，了解产品架构、功能及落地方式。

## 1.4 核心目标

- ✅ 统一第三方 AI API 接入入口，支持快速切换模型
- ✅ 构建个人专属核心能力（记忆、RAG 知识库、提示词管理）
- ✅ 采用轻量化分层架构，确保快速落地、可扩展
- ✅ 实现成本可控、安全可靠，具备基础监控能力
- ✅ 支持 Web 端接入，后续扩展多端

# 二、MVP 范围界定

## 2.1 MVP 核心原则

**聚焦核心，快速落地**：优先实现 80% 高频使用的核心功能，剩余 20% 功能在后续迭代中完善。

## 2.2 MVP 功能范围（阶段一：1-2 个月）

### ✅ 包含的功能

#### 核心功能（P0 优先级）
- **用户系统**：注册、登录、API Key 管理
- **基础对话**：支持单次问答、流式输出
- **模型路由**：支持 2-3 个模型配置、手动切换
- **会话管理**：创建会话、查看历史、删除会话
- **基础记忆**：会话级上下文记忆（最近 10 轮对话）

#### 重要功能（P1 优先级）
- **提示词模板**：预设 5-10 个常用角色模板
- **知识库 V1**：支持 Markdown/TXT 文档上传、基础检索
- **使用统计**：Token 消耗统计、请求次数统计
- **基础监控**：API 调用日志、错误日志

#### 可选功能（P2 优先级）
- **简单路由策略**：成本优先/性能优先切换
- **配额管理**：每日请求限额告警

### ❌ 不包含的功能（后续迭代）

#### 阶段二（3-4 个月）
- ❌ 复杂工作流引擎
- ❌ 可视化流程编排
- ❌ 多端接入（桌面端、移动端、插件）
- ❌ 长期人格记忆系统
- ❌ 高级 RAG（混合检索、重排序）

#### 阶段三（5-6 个月）
- ❌ 场景化模板系统
- ❌ 提示词版本管理
- ❌ 故障自动熔断切换
- ❌ 团队协作功能

## 2.3 MVP 与非 MVP 对比

| 功能模块 | MVP 版本 | 完整版（后续） |
|---------|---------|---------------|
| **接入端** | Web 端 | Web+ 桌面 + 移动端 + 插件 |
| **认证方式** | 账号密码+API Key | +OAuth2、SSO |
| **对话能力** | 基础问答 + 流式 | + 工作流编排 |
| **记忆系统** | 会话级短期记忆 | + 长期人格记忆 |
| **知识库** | 单文档上传+ 基础检索 | 批量上传 + 混合检索 + 重排序 |
| **提示词** | 预设模板 | + 版本管理 + 自动优化 |
| **模型路由** | 手动切换 + 简单策略 | + 智能路由 + 自动熔断 |
| **监控告警** | 基础日志 + 统计 | + 实时告警 + 链路追踪 |
| **部署方式** | 单机 Docker | + K8s + 高可用 |

# 三、简化分层架构（MVP 版 4 层）

## 3.1 架构说明

MVP 版本简化为 4 层架构，聚焦核心功能，降低落地复杂度：

```
┌─────────────────────────────────┐
│   第一层：接入应用层（Web 端）    │
└──────────────┬──────────────────┘
               │
┌──────────────▼──────────────────┐
│   第二层：业务逻辑层             │
│   （对话/记忆/知识库/路由）      │
└──────────────┬──────────────────┘
               │
┌──────────────▼──────────────────┐
│   第三层：模型适配层             │
│   （统一接口 + Provider 适配）    │
└──────────────┬──────────────────┘
               │
┌──────────────▼──────────────────┐
│   第四层：基础设施层             │
│   （数据库 + 缓存 + 日志）        │
└─────────────────────────────────┘
```

## 3.2 各层职责

### 第一层：接入应用层（Web 端）
- Web 管理后台（Vue3 + Element Plus）
- 统一 API 接入入口
- 基础 CORS 配置

### 第二层：业务逻辑层
- **对话服务**：处理用户提问、组织上下文
- **记忆服务**：管理会话级上下文
- **知识库服务**：文档解析、向量检索
- **路由服务**：模型选择策略

### 第三层：模型适配层
- 统一模型接口标准
- Provider 适配器（OpenAI、Claude、DeepSeek）
- 协议转换、流式输出适配

### 第四层：基础设施层
- PostgreSQL + pgvector（关系 + 向量存储）
- Redis（缓存、限流）
- 结构化日志
- 基础监控指标

# 四、核心功能详细设计

## 4.1 用户系统

### 4.1.1 功能描述
提供用户注册、登录、API Key 管理功能。

### 4.1.2 功能点
- 用户注册（用户名 + 邮箱 + 密码）
- 用户登录（JWT Token）
- API Key 生成、撤销、查看
- 用户信息修改

### 4.1.3 业务流程
```
注册：填写信息 → 校验格式 → 创建用户 → 返回 Token
登录：输入账号密码 → 验证 → 返回 Token
创建 API Key：验证 Token → 生成 Key → 哈希存储 → 返回明文（仅一次）
```

## 4.2 对话系统

### 4.2.1 功能描述
提供基础 AI 对话能力，支持流式输出。

### 4.2.2 功能点
- 创建对话请求（支持流式/非流式）
- 会话上下文管理（最近 10 轮）
- 模型手动切换
- 会话历史查看

### 4.2.3 对话流程
```
用户提问
  ↓
加载会话历史（最近 10 轮）
  ↓
拼接提示词模板
  ↓
选择模型
  ↓
调用 Provider API
  ↓
流式返回/一次性返回
  ↓
保存对话记录
```

## 4.3 知识库系统（V1）

### 4.3.1 功能描述
支持个人文档上传、向量化、检索。

### 4.3.2 功能点
- 创建知识库
- 上传文档（Markdown/TXT）
- 文档自动分块、向量化
- 语义检索（Top 5）
- 知识库删除

### 4.3.3 文档处理流程
```
上传文档
  ↓
解析内容（提取文本）
  ↓
分块（500-1000 tokens，重叠 100）
  ↓
调用 Embedding 模型
  ↓
存储向量到 pgvector
  ↓
建立索引
```

### 4.3.4 检索流程
```
用户提问
  ↓
生成问题向量
  ↓
向量相似度搜索（Top 5）
  ↓
返回相关文档片段
  ↓
拼接到 Prompt 中
```

## 4.4 记忆系统（会话级）

### 4.4.1 功能描述
管理当前会话的上下文记忆。

### 4.4.2 功能点
- 自动保存对话历史
- 上下文窗口管理（最近 10 轮）
- 超额自动截断
- 会话结束自动归档

### 4.4.3 记忆策略
```
- 临时记忆：当前对话有效，关闭页面丢失
- 会话记忆：会话周期内有效，最多 10 轮
- 永久记忆：暂不实现（阶段二）
```

## 4.5 模型路由（基础版）

### 4.5.1 功能描述
支持手动选择模型，预设简单路由策略。

### 4.5.2 功能点
- 模型列表查看
- 手动切换模型
- 默认模型设置
- 简单路由策略（成本优先/性能优先）

### 4.5.3 支持的模型（MVP）
- OpenAI GPT-3.5-turbo
- OpenAI GPT-4
- DeepSeek Chat
- （后续扩展：Claude、通义千问）

## 4.6 提示词模板

### 4.6.1 功能描述
预设常用角色模板，快速切换 AI 角色。

### 4.6.2 预设模板（10 个）
1. 通用助手（默认）
2. 编程助手
3. 文案创作
4. 翻译助手
5. 学习导师
6. 代码审查员
7. 技术顾问
8. 数据分析
9. 思维导图
10. 会议纪要

### 4.6.3 模板结构
```
角色：{role_name}
目标：{goal}
风格：{style}
约束：{constraints}
示例：{examples}
```

## 4.7 统计与监控

### 4.7.1 功能描述
基础使用统计和监控日志。

### 4.7.2 统计功能
- Token 消耗统计（按日/按模型）
- 请求次数统计
- 会话数量统计

### 4.7.3 监控功能
- API 调用日志
- 错误日志
- 响应时间统计

| 指标 | 目标值 | 说明 |
|------|-------|------|

## 6.2 可用性指标

| 指标 | 目标值 | 说明 |
|------|-------|------|
| 服务可用性 | 99.5% | 允许短暂停机 |
| 数据持久化 | 99.9% | 数据不丢失 |
| 备份频率 | 每日 1 次 | 自动备份 |

## 6.3 成本预算

| 项目 | 预算 | 说明 |
|------|------|------|
| 云服务器 | < ¥300/月 | 2 核 4G |
| 第三方 API | < ¥2000/月 | 可控 |
| 域名 SSL | < ¥100/年 | 可选 |
