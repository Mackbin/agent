> **你的处境**：前端深度（Vue3/TS/企微SDK/WebSocket/组件脚手架），但感到被动，担心被后端和AI双重挤压。
>
> **你的目标**：通过中医AI诊所这个实战项目，完成从前端到「全栈+AI应用工程师」的能力转型，让自己成为**不可替代的产品交付者**。
>
> **本方案的核心逻辑**：不是对抗AI，而是**成为AI的驾驭者**——让AI帮你补齐后端短板，让你一个人能干原来一个团队的活。

## 一、先直面现实：前端为什么会被替代？

### 1.1 真实威胁分析

```

├─────────────────────────────────────────────────────────────┤
```

### 1.2 但你有一个巨大的机会窗口

```
❌ 被替代的前端特征：
   - 只会写 UI，不懂业务逻辑
   - 依赖设计稿，不会独立思考功能
   - 不懂数据怎么存、API 怎么设计
   - 把AI当成威胁，而不是工具

✅ 不可替代的前端特征（你要成为的人）：
   - 能独立设计并交付一个完整产品（前后端+数据库）
   - 懂业务场景，知道做什么功能能解决真实痛点
   - 能用AI加速自己的工作效率（10倍效率）
   - 有垂直行业认知（比如中医），这是纯技术人没有的优势
```

**关键洞察**：

> AI 不会替代前端工程师，但**会用AI的前端工程师会替代不会用的**。
> 
> 同样，**能独立交付产品的全栈工程师会替代只能写页面的前端**。
> 
> 你的机会在于：**你有前端深度 + 你已经开始写后端了 + 你有真实场景（朋友诊所）**

## 四、🧠 Layer 4：全栈 AI 层 — 你的核心竞争力

### 4.1 为什么这一层最重要？

```
传统前端 vs AI应用工程师：

传统前端的工作方式：
  设计稿 → 写HTML/CSS/JS → 调样式 → 提交PR
  → 这个过程AI已经能做到80%了

AI应用工程师的工作方式：
  业务需求 → 设计Prompt → 接入LLM API → 构建RAG知识库
  → 编排Agent工作流 → 部署上线 → 监控效果
  → 这个过程AI做不到，因为需要：
     ① 理解业务场景（AI不理解）
     ② 判断什么该用AI、什么不该用（AI不判断）
     ③ 处理AI输出的不稳定性和幻觉（AI不自省）
     ④ 整合前后端、数据库、缓存等完整链路（AI只生成片段）
```

**这就是你的护城河**：你不是在写代码，而是在**设计和控制一个AI系统**。

### 4.2 能力 PRD

| 能力域 | 定义 | 产出物 | 与你现有能力的连接点 |
|--------|------|--------|-------------------|
| **Prompt 工程** | 设计System Prompt让LLM输出稳定可控的结果 | Prompt模板库、评测报告 | 你简历中的「上下文感知与清洗」可直接迁移 |
| **RAG 检索增强** | 将领域知识向量化入库，让LLM基于真实数据回答 | 知识库、检索管道 | 你的PostgreSQL基础 + pgvector扩展 |
| **Agent 编排** | 设计多步骤AI工作流（思考→检索→判断→执行） | Agent定义、工作流DAG | 你的Workflow流程组件经验可迁移 |
| **Function Calling** | 让LLM调用你的API（查询患者档案、预约时间） | Tool定义、调用链路 | 你的FastAPI后端就是被调用的对象 |
| **Embedding & 向量检索** | 将文本转为向量，支持语义相似搜索 | Embedding服务、向量索引 | pgvector + SQLAlchemy |
| **多模态理解** | 处理图片（舌诊）、语音（问诊）、文本 | 多模态管道 | 你的ASR/TTS方案 + GPT-4V |
| **模型评测** | 定量评估不同模型/Prompt的输出质量 | 评测集、评分体系 | 你的结构化日志可支撑数据收集 |
| **安全护栏** | 防止LLM输出越界内容、医疗建议违规 | 安全规则、过滤器 | 你的认证中间件可扩展 |

```python
# 你需要在 redis.py 中实现的缓存装饰器

import json
import hashlib
from functools import wraps
from app.redis import redis_client

def cache_result(ttl: int = 300, prefix: str = "cache"):
    """
    通用缓存装饰器
    ttl: 过期时间（秒）
    prefix: 缓存键前缀
    """
    def decorator(func):
        @wraps(func)
        async def wrapper(*args, **kwargs):
            # 生成缓存键（基于函数名+参数）
            cache_key = f"{prefix}:{func.__name__}:{hashlib.md5(json.dumps(kwargs).encode()).hexdigest()}"
            
            # 尝试从缓存获取
            cached = await redis_client.get(cache_key)
            if cached:
                return json.loads(cached)
            
            # 缓存未命中，执行原函数
            result = await func(*args, **kwargs)
            
            # 写入缓存
            await redis_client.setex(cache_key, ttl, json.dumps(result))
            
            return result
        return wrapper
    return decorator


# 使用示例：缓存知识库检索结果
@cache_result(ttl=60, prefix="rag_search")
async def search_knowledge(query: str, top_k: int = 5):
    """带缓存的RAG检索"""
    embedding = await generate_embedding(query)
    results = await pgvector_search(embedding, top_k=top_k)
    return results
```

```python
# 问题1：缓存穿透（查询不存在的key，每次都打到DB）
# 解决：布隆过滤器或空值缓存
async def get_with_null_cache(key: str):
    cached = await redis.get(key)
    if cached is not None:
        if cached == "NULL":
            return None  # 空值标记
        return json.loads(cached)
    
    # 查DB
    result = await db_query(key)
    
    if result is None:
        # 缓存空值，短TTL
        await redis.setex(key, 60, "NULL")
    else:
        await redis.setex(key, 3600, json.dumps(result))
    
    return result


# 问题2：缓存击穿（热点key过期瞬间大量请求打DB）
# 解决：分布式锁 + 逻辑过期
async def get_with_lock(key: str):
    cached = await redis.get(key)
    if cached:
        return json.loads(cached)
    
    # 尝试获取锁
    lock_key = f"lock:{key}"
    lock_acquired = await redis.setnx(lock_key, "1", ex=10)
    
    if lock_acquired:
        # 获取到锁，查DB并更新缓存
        result = await db_query(key)
        await redis.setex(key, 3600, json.dumps(result))
        await redis.delete(lock_key)
        return result
    else:
        # 未获取到锁，短暂等待后重试
        await asyncio.sleep(0.05)
        return await get_with_lock(key)


# 问题3：缓存雪崩（大量key同时过期）
# 解决：随机TTL + 多级缓存
import random

ttl = 3600 + random.randint(0, 300)  # 基础TTL + 随机偏移
```

```python
from typing import Optional

class TokenTracker:
    """LLM API 调用费用追踪"""
    
    def __init__(self):
        self.total_tokens = 0
        self.total_cost = 0.0
        self.call_count = 0
        
        # 各模型的单价（每1000 token）
        self.model_pricing = {
            "gpt-4": {"input": 0.03, "output": 0.06},
            "gpt-4o": {"input": 0.005, "output": 0.015},
            "gpt-3.5-turbo": {"input": 0.0005, "output": 0.0015},
            "claude-3-opus": {"input": 0.015, "output": 0.075},
            "deepseek-chat": {"input": 0.00014, "output": 0.00028},
        }
    
    def track_call(self, model: str, input_tokens: int, output_tokens: int):
        """记录一次API调用"""
        pricing = self.model_pricing.get(model, {"input": 0.001, "output": 0.001})
        
        input_cost = (input_tokens / 1000) * pricing["input"]
        output_cost = (output_tokens / 1000) * pricing["output"]
        
        self.total_tokens += input_tokens + output_tokens
        self.total_cost += input_cost + output_cost
        self.call_count += 1
        
        logger.info(
            f"LLM_API_CALL | model={model} | "
            f"input_tokens={input_tokens} | output_tokens={output_tokens} | "
            f"cost=${input_cost + output_cost:.6f} | "
            f"total_cost=${self.total_cost:.4f} | calls={self.call_count}"
        )
    
    def get_stats(self) -> dict:
        """获取统计信息"""
        return {
            "total_tokens": self.total_tokens,
            "total_cost": round(self.total_cost, 4),
            "call_count": self.call_count,
            "avg_cost_per_call": round(self.total_cost / max(self.call_count, 1), 6),
```

```vue
async function sendMessage() {
  if (!inputText.value || isStreaming.value) return
  
  const userMessage = inputText.value
  inputText.value = ''
  
  // 添加用户消息
  messages.value.push({
    id: Date.now(),
    role: 'user',
    content: userMessage,
    isStreaming: false,
  })
  
  // 添加AI占位消息
  const assistantMessageId = Date.now() + 1

    id: assistantMessageId,
    role: 'assistant',
    content: '',
    isStreaming: true,
  })

    const response = await fetch('/api/v1/chat/stream', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ message: userMessage }),
      signal: abortController.signal,
    })
    
    const reader = response.body!.getReader()
    const decoder = new TextDecoder()
    let buffer = ''
    
    while (true) {
      const { done, value } = await reader.read()
      if (done) break
      
      buffer += decoder.decode(value, { stream: true })
      const lines = buffer.split('\n')
      buffer = lines.pop() || ''

        if (line.startsWith('data: ')) {
          const data = line.slice(6)

            finishStream(assistantMessageId)
            return
          }

            const parsed = JSON.parse(data)
            const msg = messages.value.find(m => m.id === assistantMessageId)
            if (msg) {
              msg.content += parsed.content || ''
              scrollToBottom()
            }
          } catch (e) {
            // 忽略解析错误
          }
        }
      }

    if ((error as Error).name === 'AbortError') {
      console.log('用户停止生成')
    } else {
      console.error('流式接收错误:', error)
    }
    finishStream(assistantMessageId)
  }
}

    msg.isStreaming = false
  }
}

      top: messageListRef.value.scrollHeight,
      behavior: 'smooth',
```

## 九、📊 数据工程 — 被遗漏的关键板块

### 9.1 为什么数据工程对你的项目至关重要？

```
RAG的效果 = 数据质量 × 检索算法 × Prompt质量

如果你的数据是一坨垃圾：
  - 手写病历，字迹潦草
  - PDF扫描件，OCR错误率高
  - 格式混乱，有的按时间排序有的按症候
  - 大量缩写和口语化表达

那再好的算法也救不回来。

→ 数据工程决定了你RAG的上限
```

### 9.2 中医数据处理管线

```
原始数据 → 清洗 → 结构化 → 分块 → 向量化 → 入库

Step 1：数据采集
├── 来源1：纸质病历拍照 → OCR识别
├── 来源2：电子病历导出（Excel/CSV）
├── 来源3：医生口述录音 → ASR转文字
└── 来源4：教材/论文PDF → 文本提取

Step 2：数据清洗
├── 去除个人信息（姓名、电话、地址）→ 脱敏
├── 统一术语（"苔腻"="舌苔厚腻"="舌质胖大苔白腻"）
├── 去除乱码和特殊字符
└── 格式标准化（统一JSON/Markdown格式）

Step 3：结构化（可选，提升检索精度）
├── 提取字段：症状、诊断、方剂、医嘱
├── 建立标签体系：内科/外科/妇科/儿科...
└── 关联关系：症状↔诊断↔方剂

Step 4：分块（Chunking）
├── 按段落切分（自然边界）
├── 每块200-500字（平衡粒度和语义完整性）
├── 保留上下文窗口（相邻块有重叠）
└── 元数据标注（来源、类别、日期）

Step 5：向量化
├── 调用Embedding API
├── 生成1536维向量（ada-002）
└── 存入pgvector

Step 6：入库 + 索引
├── 创建IVFFlat/HNSW索引
├── 设置召回参数（lists, probes）
└── 验证检索效果
```

```python
# app/services/data_pipeline.py

import re
import json
from typing import List, Dict
from pathlib import Path

class TCMDataCleaner:
    """中医数据清洗器"""
    
    # 中医同义词映射表（持续扩充）
    SYNONYMS = {
        "苔腻": ["舌苔厚腻", "舌质胖大苔白腻", "苔白腻", "苔黄腻"],
        "脉弦": ["弦脉", "脉弦细", "左关弦"],
        "气虚": ["脾气虚", "肺气虚", "肾气虚"],
        "阴虚": ["肾阴虚", "肝阴虚", "胃阴虚"],
        # ... 更多同义词
    }
    
    # 个人信息正则（脱敏用）
    PII_PATTERNS = {
        "phone": re.compile(r'1[3-9]\d{9}'),
        "id_card": re.compile(r'\d{17}[\dXx]'),
        "name": re.compile(r'(?:患者|病人|姓名)[：:]\s*([\u4e00-\u9fa5]{2,4})'),
    }
    
    def clean_text(self, text: str) -> str:
        """清洗单条文本"""
        # 1. 脱敏
        text = self._remove_pii(text)
        
        # 2. 统一术语
        text = self._normalize_terminology(text)
        
        # 3. 去除特殊字符
        text = re.sub(r'[^\u4e00-\u9fa5a-zA-Z0-9，。；：""''！？、（）\s]', '', text)
        
        # 4. 去除多余空白
        text = re.sub(r'\s+', ' ', text).strip()
        
        return text
    
    def _remove_pii(self, text: str) -> str:
        """移除个人隐私信息"""
        for pattern in self.PII_PATTERNS.values():
            text = pattern.sub('[已脱敏]', text)
        return text
    
    def _normalize_terminology(self, text: str) -> str:
        """统一中医术语"""
        for standard, variants in self.SYNONYMS.items():
            for variant in variants:
                text = text.replace(variant, standard)
        return text


class TextChunker:
    """文本分块器"""
    
    def __init__(self, chunk_size: int = 400, overlap: int = 50):
        self.chunk_size = chunk_size
        self.overlap = overlap
    
    def chunk(self, text: str) -> List[Dict]:
        """
        智能分块：优先按段落切分，超长段落再按句子切分
        """
        chunks = []
        
        # 按段落分割
        paragraphs = re.split(r'\n\s*\n', text)
        
        current_chunk = ""
        
        for para in paragraphs:
            para = para.strip()
            if not para:
                continue
            
            # 如果当前块加上新段落不超过大小限制，直接拼接
            if len(current_chunk) + len(para) <= self.chunk_size:
                current_chunk += ("\n" if current_chunk else "") + para
            else:
                # 当前块已满，保存
                if current_chunk:
                    chunks.append({"content": current_chunk, "length": len(current_chunk)})
                
                # 新段落本身超过限制，需要进一步切分
                if len(para) > self.chunk_size:
                    sub_chunks = self._split_long_text(para)
                    chunks.extend(sub_chunks)
                    current_chunk = ""
                else:
                    current_chunk = para
        
        # 最后一块
        if current_chunk:
            chunks.append({"content": current_chunk, "length": len(current_chunk)})
        
        # 添加元数据
        for i, chunk in enumerate(chunks):
            chunk["index"] = i
            chunk["total"] = len(chunks)
        
        return chunks
    
    def _split_long_text(self, text: str) -> List[Dict]:
        """切分超长文本"""
        sentences = re.split(r'(?<=[。！？])', text)
        chunks = []
        current = ""
        
        for sent in sentences:
            if len(current) + len(sent) > self.chunk_size:
                if current:
                    chunks.append({"content": current, "length": len(current)})
                current = sent
            else:
                current += sent
        
        if current:
            chunks.append({"content": current, "length": len(current)})
        
        return chunks


# 使用示例
if __name__ == "__main__":
    cleaner = TCMDataCleaner()
    chunker = TextChunker(chunk_size=400, overlap=50)
    
    # 模拟一条原始医案

    患者：张三，男，45岁，电话13812345678
    主诉：失眠3个月，加重1周
    现病史：患者3个月前因工作压力大开始出现入睡困难，
    近1周加重，每晚需1-2小时才能入睡，多梦易醒，
    醒后难以再睡，伴头晕耳鸣，腰膝酸软，手足心热。
    舌质红，苔薄黄，脉弦细数。
    诊断：不寐（肝肾阴虚证）
    方药：酸枣仁汤加减
    医嘱：忌辛辣刺激，按时作息
    """
    
    # 清洗
    cleaned = cleaner.clean_text(raw_text)
    print("清洗后:", cleaned[:100])
    
    # 分块
    chunks = chunker.chunk(cleaned)
    print(f"\n分块结果：共{len(chunks)}块")
    for c in chunks:
        print(f"  [{c['index']}] ({c['length']}字) {c['content'][:50]}...")
```

```python
class SafetyGuardrails:
    """安全护栏"""
    
    FORBIDDEN_PATTERNS = [
        r"确诊为.*癌",
        r"你得了.*病",
        r"建议服用.*毫克",
        r"我可以给你开.*药方",
        r"这是.*的特效药",
    ]
    
    @staticmethod
    def add_disclaimer(response: str) -> str:
        """在回答末尾追加免责声明"""
        if not response.endswith("为准"):
            return response + SAFETY_DISCLAIMER
        return response
    
    @staticmethod
    def check_safety(response: str) -> tuple[bool, str]:
        """
        检查输出安全性
        返回：(是否安全, 处理后的内容或原因)
        """
        import re
        
        for pattern in SafetyGuardrails.FORBIDDEN_PATTERNS:
            if re.search(pattern, response):
                # 重写危险内容
                safe_response = (
                    "抱歉，我无法为您提供具体的诊断或用药建议。\n"
                    "中医诊疗需要结合您的具体情况由专业医师面诊。\n"
                    "建议您前往正规医疗机构就诊。" 
                    + SAFETY_DISCLAIMER
                )
                return False, safe_response
        
        return True, SafetyGuardrails.add_disclaimer(response)
```
