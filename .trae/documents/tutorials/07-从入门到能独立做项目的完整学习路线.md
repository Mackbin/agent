```
  - 接收文本 → 分块 → 向量化 → 存入 pgvector
  - 返回：已导入 N 个文本块

▸ 第三步：改造 chat 接口（3小时）
  - 在调用 LLM 之前，先检索知识库
  - 把检索结果作为 context 注入到 system prompt
  - 要求 LLM 必须基于 context 回答

▸ 第四步：验证效果（2小时）
  - 问一个知识库里有的问题 → AI 基于资料回答 ✅
  - 问一个知识库里没有的问题 → AI 说不知道 ✅
  - 问一个需要编造的问题 → AI 不编造 ✅

▸ 第五步：流式输出（2小时）
  - 确保检索 + LLM 生成 + 流式推送给前端全链路跑通
  - 前端能收到逐字生成的内容
```

### 关键习惯建立

```
1. 每天都要 git commit
   即使只改了一行代码，也要提交
   
2. 每周回顾一次
   打开你的 git log，看看这周改了什么
   
3. 遇到报错先自己排查
   → 读报错信息（Python 的报错比 JS 清晰很多）
   → 用 print() 打印中间变量
   → 还解决不了再搜索
   
4. 用 AI 辅助，但不是让 AI 替你写
   → 正确用法：让 AI 解释你不懂的代码
   → 错误用法：让 AI 生成全部代码后直接粘贴
```
